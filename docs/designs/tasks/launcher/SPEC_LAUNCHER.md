## Launcher Page(Home page)
Purpose: show recent projects and start a new project.

### main interface
#### Visual effect
![dark](../../visuals/Launcher-dark.png)
![light](../../visuals/Launcher-light.png)

#### Primary layout:
Header
- left side filled with `Recent projects` title, and the `Local workspace` at top of title .
- right side is `New project` button.

Body:
- Recent project list.
- pagination

#### Interaction detail
1. Buttons:

- `New project`: get into the Setup subflow.
- Project card: click the entire card opens `Editor` page for that project.

2. Project card fields:

- thumbnail.
- project name.
- voice duration.
- sentence count.
- media count.
- last rendering time, if it doesn't have, hide it.
- show last render of the project status tag, if it doesn't have, hide it.

3. Thumbnail rules:

- Prefer the first frame of the latest successful rendered video.
- If no rendered video exists, generate a deterministic placeholder image when the project is created: three random colors covered evenly. Store or cache the generated thumbnail path.
- Click the thumbnail's play icon, open a modal to display the rendered video, check ![dark effect](../../visuals/Launcher-play-dark.png) and ![light effect](../../visuals/Launcher-play-light.png)

Launcher states:

- No recent projects.
- Project config missing/corrupt in database. (delete it)
- tag status: `unrendered`, `queued`, `rendering`, `rendered`, `failed`, `cancelled`.
- pagination page size, page index

## API Surface Required By Prototype

Launcher:

- `GET /projects`
- `DELETE /projects/:projectId`

Setup:

- `POST /subtitle`: generate a `subtitle.srt` for the `voice` file
- `POST /subtitle/alignment`: adjust the accuracy of the `subtitle.srt` according to `transcript` file provided by user
- `POST /projects`: create a new project, also including generate the fallback thumbnai of the project

## Edge Cases And Boundary Conditions

Setup and alignment:

- Voice missing.
- Transcript missing.
- Voice codec unsupported.
- Transcript empty.
- Transcript has one very long paragraph.
- Transcript segmentation differs from user expectation.
- WhisperX not installed.
- CUDA unavailable.
- CUDA OOM falls back to CPU or fails clearly.
- Alignment fails due long silence or mismatched text.
- User edits voice or transcript after assigning clips.


## Testing Strategy

Shared fixtures:

- `test01` ([../../projects/test01](../../projects/test01)) is the canonical E2E/integration fixture with voice, transcript, mixed media, and a project config that exercises every layer type.

### Frontend unit/component tests (Vitest + Testing Library)

Launcher (Home):

- Recents grid sorts by `last_rendered` descending.
- Empty state renders the `New project` CTA when no recents exist.
- Project card renders thumbnail + name + voice duration + sentence count + media count + last rendering time + status tag; never renders the raw folder path.
- Thumbnail falls back per spec rules: first frame of the latest successful render -> deterministic three-color placeholder.
- `New project` starts the Setup subflow directly.
- Clicking the thumbnail play icon plays the video of the project.
- Pagination at bottom behaves correctly when switching page index.

Setup subflow (4-step flow):

- Header `Cancel` returns to Launcher without persisting partial state.
- Step 1 - Project Name: typing a name checks the step; clearing un-checks it.
- Step 1 - Output Preset segmented control offers `720p (Draft)`, `1080p (Final)`, `1080p / vertical (9:16)`; selection persists into the new project's config.
- Step 2 - Voice card: clicking opens the OS file picker; accepts `.mp3 / .wav / .m4a` only; an unsupported file surfaces a non-blocking error.
- Step 2 - Voice card shows a green border once a valid voice is selected.
- Step 3 - Subtitle Generate panel: `Generate subtitle` button is disabled until steps 1+2 are checked. Status cycles `ready -> running -> succeeded / failed`. On success the panel displays the cue count and total duration of the generated `subtitles.srt`.
- Step 3 - `Subtitle` is checked only when `subtitles.srt` has been generated successfully from the voice.
- Step 4 - Alignment panel: trigger button is disabled until steps 1+2+3 are checked. Status cycles `ready -> running -> succeeded / failed`. On success the panel reports the number of corrections applied to `subtitles.srt`.
- Step 4 - `Transcript for alignment` file picker accepts the transcript; `Watermark for video` is optional and uses the same picker pattern.
- Step 4 - `Alignment` is checked only when subtitle alignment runs successfully.
- `Create project` is disabled unless all four steps are checked. Clicking it creates the project and navigates to `/editor/:projectId`.
- State coverage: project name provided/missing; voice selected/missing/invalid; transcript selected/missing/empty/invalid; watermark absent/selected; Subtitle Generate ready/running/succeeded/failed; Subtitle Alignment ready/running/succeeded/failed.

### Backend tests (pytest)

Subtitle Generate (voice -> `subtitles.srt`, runs from Setup step 3):

- Accepts `.mp3 / .wav / .m4a`; rejects unsupported codecs with a clear error.
- Status transitions `ready -> running -> succeeded` are emitted on the WS; failure emits `failed` and is recoverable.
- Output `subtitles.srt` cue durations sum to the voice duration within +/-200 ms.
- Re-running with an unchanged voice hash is a cache hit with no model re-load.

Subtitle Alignment (transcript + voice + `subtitles.srt` -> corrected `subtitles.srt`, runs from Setup step 4):

- WhisperX reference-text mode aligns each sentence within 50 ms of ground truth on the fixture voice.
- The number of corrections reported by the alignment matches the cue-level diff between generated and aligned `subtitles.srt`.
- Changing transcript text or voice file hash invalidates `.vc/alignment.json` and re-runs alignment.
- WhisperX-missing / CUDA-OOM / CUDA-unavailable surface as recoverable errors and fall back to CPU when possible.

Launcher/Setup API surface:

- `GET /projects` returns Launcher recents.
- `DELETE /projects/:projectId` removes a recent project entry.
- `POST /subtitle` runs Subtitle Generate.
- `POST /subtitle/alignment` runs Subtitle Alignment.
- `POST /projects` creates a project only after Setup succeeds and generates the fallback thumbnail.

### E2E / browser tests (Playwright)

Use fixture `test01` and deterministic API mocks for each flow. Each flow is one Playwright `test()` case and executes full user paths, not isolated assertions.

Flow 1 - Launcher empty state to Setup entry:

1. Mock `GET /projects` -> `[]`; open `/launcher`.
2. Assert `Local workspace`, `Recent projects`, `New project` visible and no project card exists.
3. Click `New project`.
4. Assert URL is Setup route and Setup header (`New project` + `SetUp`) is visible.
5. Click `Cancel`.
6. Assert back on Launcher and `POST /projects` was never called.

Flow 2 - Launcher populated state, pagination, preview modal, editor navigation:

1. Mock `GET /projects` with more than one page and mixed statuses (`unrendered`, `rendering`, `rendered`, `failed`).
2. Open `/launcher`; assert cards on page 1 are sorted by `last_rendered` descending.
3. On first rendered card, assert visible fields: thumbnail, name, voice duration, sentence count, media count; assert raw folder path text is absent.
4. Click thumbnail play icon; assert modal opens and video element is playable; close modal and assert focus returns to Launcher content.
5. Navigate to page 2 and back to page 1; assert page switch changes visible card set but keeps ordering rule.
6. Click a project card body (not play icon); assert navigation to `/editor/:projectId`.

Flow 3 - Setup happy path from start to project creation:

1. Enter Setup from Launcher (`New project` button).
2. Assert all four checklist steps are initially unchecked and `Create project` is disabled.
3. Step 1: fill project name and choose output preset (`1080p (Final)`); assert step 1 checked.
4. Step 2: upload valid voice file (`.wav`); assert voice card success style (green border) and step 2 checked.
5. Step 3: click `Generate subtitle`; assert status transitions `ready -> running -> succeeded`; assert cue count + duration summary rendered; assert step 3 checked.
6. Step 4: upload transcript, leave watermark empty, run alignment; assert `ready -> running -> succeeded` and correction count rendered; assert step 4 checked.
7. Assert `Create project` becomes enabled only after step 4 success.
8. Click `Create project`; assert one `POST /projects` call with selected preset and navigate to `/editor/:projectId`.

Flow 4 - Setup validation/failure recovery:

1. Enter Setup and complete step 1.
2. Upload unsupported voice file type; assert recoverable error UI, step 2 remains unchecked, subtitle button disabled.
3. Replace with valid voice and run subtitle generation with mocked first failure then success:
   - first run returns failure -> step 3 unchecked, error shown, retry still enabled.
   - second run returns success -> step 3 checked.
4. Run alignment with mocked first failure then success:
   - first run returns failure -> step 4 unchecked.
   - second run returns success -> step 4 checked.
5. Assert flow can continue to enabled `Create project` without page reload.

Flow 5 - Dependency reset behavior after upstream input changes:

1. Reach a fully successful state (all four steps checked).
2. Replace voice file with a different valid file.
3. Assert `Subtitle` and `Alignment` steps reset to unchecked and `Create project` becomes disabled.
4. Re-run subtitle + alignment to restore both checks.
5. Change transcript file.
6. Assert only alignment-dependent state resets (`Alignment` unchecked, `Subtitle` remains checked).
7. Re-run alignment and assert `Create project` re-enables.

Flow 6 - Theme/actionability smoke (dark and light):

1. Run Flow 1 and Flow 3 once in dark theme and once in light theme.
2. In each theme run, verify critical controls are clickable and visible: `New project`, `Cancel`, voice picker, subtitle trigger, alignment trigger, `Create project`.
3. Save screenshots for Launcher and Setup checkpoints for parity baselines.

## Success Criteria

Phase 1 Launcher and Setup work is accepted when all items below hold.

### Functional Acceptance

Launcher:

- The app opens to a Launcher home with a recents grid that persists across restarts.
- Picking `New project` enters the Setup subflow directly.
- Each project card displays thumbnail + name + voice duration + sentence count + media count + last rendering time + status tag.
- The raw folder path is never displayed in the primary Launcher UI.
- The thumbnail's play icon plays the project's latest successful render.
- If no rendered video exists, the project thumbnail uses the deterministic three-color placeholder generated when the project is created.

Setup:

- The on-disk project layout (`voice.wav`, `transcript.txt`, `subtitles.srt`, `media/`, `renders/`, `.vc/`) is materialized only when `Create project` succeeds at the end of Setup.
- Step 1 - Project Name + Output Preset (`720p Draft` / `1080p Final` / `1080p / vertical 9:16`) are checkable.
- Selected output preset persists into the project config.
- Step 2 - Voice picker accepts `.mp3 / .wav / .m4a`; the card shows a green border on a valid selection.
- Step 3 - Subtitle Generate (voice -> `subtitles.srt`) runs on the backend; the panel reports cue count + duration on success; its button is enabled only after steps 1 and 2.
- Step 4 - Subtitle Alignment (transcript + `subtitles.srt` -> corrected `subtitles.srt`) uses the user-supplied transcript and runs WhisperX reference-text alignment; the panel reports the number of corrections applied; its button is enabled only after steps 1, 2, and 3.
- Watermark is optional and selected from disk.
- `Create project` is enabled only when all four steps are checked, and on success navigates to `/editor/:projectId`.
- `Cancel` returns to Launcher without persisting partial state.

### Performance Targets

| Surface | Target |
| --- | --- |
| Subtitle Generate (voice -> `subtitles.srt`, warm model) | <= 0.5x voice duration end-to-end |
| Subtitle Alignment (WhisperX reference-text, warm model) | <= 0.4x voice duration end-to-end |

### Quality Gates

- Setup/alignment edge cases have matching tests: voice missing, transcript missing, unsupported voice codec, empty transcript, long paragraph transcript, segmentation mismatch, WhisperX missing, CUDA unavailable, CUDA OOM, long silence/mismatched-text alignment failure, and user edits voice/transcript after assigning clips.
- User-triggered Setup failures surface non-blocking, recoverable errors with a clear next action.
- Launcher and Setup visual parity coverage exists for every embedded screenshot and dark/light state.
